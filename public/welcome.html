Panduan Cepat Lihat Endpoint

Penyangkalan

API reverse-engineer bersifat tidak stabil. Disarankan menggunakan API resmi berbayar untuk menghindari risiko pemblokiran.

Proyek ini hanya untuk riset dan pembelajaran. Tidak menerima transaksi apapun. Hanya untuk penggunaan pribadi, dilarang untuk komersial atau layanan publik.

Panduan Integrasi

01 · Mendapatkan Token Kimi (Cara 1: Fitur Lengkap)

Dapatkan refresh_token dari kimi.moonshot.cn

1. Masuk ke Kimi dan mulai percakapan apapun
2. Tekan F12 untuk membuka Developer Tools
3. Cari nilai refresh_token di Application > Local Storage

Authorization: Bearer REFRESH_TOKEN_ANDA

Fitur yang didukung: Percakapan dasar, Percakapan multi-putaran, Pencarian internet, Upload file, Analisis gambar, Percakapan agen AI, Mode eksplorasi, Model K1/K2

02 · Mendapatkan Token Kimi (Cara 2: Percakapan Dasar)

Dapatkan Cookie kimi-auth dari kimi.moonshot.cn

1. Kunjungi https://kimi.moonshot.cn dan login
2. Tekan F12 untuk membuka Developer Tools
3. Buka Application > Cookies > https://www.kimi.com
4. Cari Cookie kimi-auth
5. Salin nilai JWT token lengkap (diawali dengan eyJ)

Authorization: Bearer TOKEN_JWT_KIMI_AUTH_ANDA

Fitur yang didukung: Percakapan dasar, Respons streaming, Berbagai skenario (K2, Pencarian, Riset)
Tidak didukung: Percakapan multi-putaran, Upload file, Analisis gambar, Percakapan agen AI

03 · Pemilihan API Otomatis

Layanan akan secara otomatis memilih API yang sesuai berdasarkan jenis Token:

  • JWT Token (typ: access) → Menggunakan Connect RPC API
  • Refresh Token (typ: refresh) → Menggunakan API Tradisional

Rekomendasi:

  • Untuk fitur lengkap, gunakan API Tradisional (refresh_token)
  • Untuk percakapan dasar saja, bisa gunakan Connect RPC (kimi-auth)

04 · Rotasi Multi-Akun

Kimi membatasi akun biasa menjadi 30 putaran tanya-jawab teks panjang setiap 3 jam (teks pendek tidak dibatasi). Anda bisa menggunakan beberapa refresh_token secara bergantian:

Authorization: Bearer TOKEN1,TOKEN2,TOKEN3

Setiap request, layanan akan memilih salah satu token secara acak.

05 · Contoh Pemanggilan API

Menggunakan endpoint yang kompatibel dengan OpenAI:

curl -X POST /v1/chat/completions \
  -H "Authorization: Bearer TOKEN_ANDA" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kimi-k2",
    "messages": [
      {
        "role": "user",
        "content": "Halo"
      }
    ],
    "use_search": false,
    "stream": false
  }'
